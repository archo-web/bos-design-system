사이드바 예시 — "멤버"가 active 상태. 테마 토글로 라이트/다크 비교해 보세요.

Navigation list item 컴포넌트 스펙

Figma 컴포넌트: Navigation list item (id 1149:82699). 7개 variant.

State 배경 토큰 텍스트 토큰
Initial — (투명) text-body
Hover / Active bg-brand-softer text-fg-brand
Disabled — (투명) text-fg-disabled
다크모드 구현 주의 — Figma의 Active 스펙(`bg-brand-softer` + `text-fg-brand`)은 다크모드에서 같은 hue 동조로 가독성이 약합니다. 웹 구현 시 다크모드만 override하세요. 아래 "Selected / Active 상태" 섹션에서 라이브 비교 가능.

Selected / Active 상태 — 디자인 가이드

선택됨을 표시하는 컴포넌트(네비, 탭, 토글 등)의 표준 패턴. 테마 토글로 다크 모드를 켜고 비교해 보세요 — 차이가 명확히 보입니다.

흔한 함정 — 다크 배경 위에 같은 hue 텍스트(brand 배경 + brand 텍스트)는 색조 동조 효과로 묻혀 보입니다. WCAG 대비비 3:1 미만으로 떨어지는 경우가 많아요.

패턴 A: 진한 brand 배경 + 흰 텍스트 권장

가장 표준적. Material, Linear, GitHub 모두 이 패턴. 대비비 7:1+ (WCAG AAA).

대시보드
멤버 (active)
설정
background: var(--colors-background-bg-brand); /* brand-700 (Light) / brand-600 (Dark) */ color: var(--colors-text-text-white); /* 항상 흰색 */

패턴 B: 옅은 brand 배경 + brand 텍스트 라이트 추천

라이트 모드에선 부드러운 강조 (Notion 스타일). 다크 모드에선 자동으로 패턴 A로 전환.

대시보드
멤버 (active)
설정
/* 라이트 */ .nav--active { background: var(--colors-background-bg-brand-softer); color: var(--colors-text-text-fg-brand); } /* 다크 — 패턴 A로 전환 */ [data-theme="dark"] .nav--active { background: var(--colors-background-bg-brand); color: var(--colors-text-text-white); }
↑ 이 사이트의 사이드바 active 메뉴가 정확히 이 패턴 — 위에서 동작 확인 가능

패턴 C: 좌측 indicator + 미니멀 배경 미니멀

VSCode, Notion 사이드바 패턴. 색조 동조 문제 자체가 없음.

대시보드
멤버 (active)
설정
.nav--active { background: var(--colors-background-bg-tertiary); color: var(--colors-text-text-heading); position: relative; } .nav--active::before { content: ''; position: absolute; left: 0; top: 25%; bottom: 25%; width: 3px; background: var(--colors-background-bg-brand); border-radius: 0 2px 2px 0; }

컴포넌트별 권장 패턴

컴포넌트 권장 패턴 이유
Sidebar LNB A 또는 C 가장 자주 보는 메뉴 — 명확한 표시 필요
Top Navbar A 또는 B 현재 페이지 컨텍스트 명시
Tabs B 같은 컨텍스트 내 전환 — 부드러운 강조
Toggle (선택된 옵션) A 현재 선택 상태가 명확해야 함
Pagination (현재 페이지) A 위치 인식 명확
Dropdown (선택된 항목) B 리스트 내 선택 — 부드러움
핵심 룰 — 다크모드에서 active 상태를 디자인할 때, 배경과 텍스트가 같은 hue 계열이라면 둘 중 하나는 흰색이거나 회색이어야 합니다. 자세한 내용은 05-interaction-patterns.md §1.3 참조.
